2004-Formal Description of the cognitive process of problem solving-Prof. Wang

Posted on 2019-04-30

Paper Title: “Formal Description of the Cognitive Process of Problem Solving, Prof. Yingxu Wang, 2004”
Questions:

  1. (Very important) A cognitive model based on semantic should be developed by myself and Prof. Huang.
  2. Whether the description of problem solving is reasonable. It is hard to say.

Highlights in this paper:

  1. The problem solving process entails performing in a new situation with information acquired and knowledge learned from past situations.
  2. A problem consists of three components: (1)givens, (2)goal, and (3)operations. Givens are information available as part of the problem. Goal is defined as the desired end state of a solution to the problem. Operations are potential actions that can be executed to achieve the problem goal.
  3. A solution to a certain problem might not exist within the solver’s current solution space.
  4. the next step is to explore the problem solving approaches. Psychologists [4-6] have proposed various possible cognitive approaches to problem solving as listed below:
    • Facts – Using direct solution path without much searching effort.
    • Hill climbing – Make any move that approaches closer to the problem goal.
    • Working backward – Frequently used in solving algebra and geometry proofs.
    • Algorithm application – Using a given and well defined solution for a problem.
    • Exhaustive search – Using a systematic search for all possible solutions.
    • Analogy approach – Using previous solutions to solve existing problem.
  5. Not everyone possesses the same problem solving abilities. Expert vs. Novice Problem Solvers.
  6. For as long as psychology has been around, there is no unified idea and model towards the cognitive process of problem solving.(!!!important)
  7. The first element enables a problem solver to understand the problem by representation. The second element enables the problem solver to search for the memory for a solution. (important)
  8. In this paper, a solution is a boolean function.(my personal understanding) refers to Definition 2.

  10. RTPA mathematical description of problem solving , as shown in Fig. 2.

2003-On Information and Knowledge Representation in the Brain-Prof. Wang

Posted on 2019-04-28

Paper Title: “On Information and Knowledge Representation in the Brain, Prof. Yingxu Wang, 2002”
Questions:

  1. Why not information and knowledge representation of the brain combines the container metaphor(by other researcher) with the relational metaphor(proposed by prof. wang)?
  2. The idea of “nowledge are represented (only) by relations, i.e. connections of synapses between neurons” is very important and can give me some insights.
  3. (Important!!!) Model neural network to store information and knowledge. Neural network can be not only used for computation and learning, but also for I and K storation.
  4. I am surperised at the capacity of our human brain, compared to the capacity of memory of modern computer.

Highlights in this paper:

  1. This paper develops the object-attribute-relation (OAR) model for describing information representation and storage in the brain.
  2. According to the OAR model, the human memory and knowledge are represented by relations, i.e. connections of synapses between neurons, rather than by the neurons themselves which is the traditional container metaphor.
  3. Based on the OAR model, the memory capacity of the human brain is calculated as in the order of 108,432 bits.
  4. The number of neurons in an adult brain has been identified to be in the order of 100 billion, and each neuron is connected to a large number of other neurons via several hundreds to a few thousands synapses.
  5. The magnitude of memory capacity of human brains is still a mystery. This is mainly because the estimation of this factor is highly dependent on suitable cognitive and mathematic models of the brain, particularly how information and knowledge are represented and stored in the memory.
  6. Comparing the human brain and those of other animals, the magnitude of the human memory shows a significant advantage.
  7. The functional model of LTM can be described as a set of (!!!hierarchical neural clusters) with partially connected neurons via synapses, as shown in Fig. 1.
  8. Conventionally, LTM is perceived as static and fixed in adult brains, This was based on the observation that the capacity of adult brains has already reached a stable state and would not grow continuously.
  9. LTM is dynamically reconfiguring, particularly at the lower levels of the neural clusters.
  10. An illustration of the OAR model between two objects is shown in Fig. 2.
  11. the memory capacity problem in cognitive science and neuropsychology can be reduced to a classical combinatorial problem, with the total potential relational combinations, among all neurons (n = 100 billion ) and their average synapses (m = 1000).
    $$C^m_n = \begin{pmatrix}n\\m\end{pmatrix} = \frac{n!}{m!(n-m)!} = 10^{8432}$$
  12. $10^{8432}$ bits. The magnitude of the brain capacity is believed as one of the profound advantages of human beings, which forms the quantitative foundation of the natural intelligence.
  13. The brain does not create new neurons to represent new information, instead it generates new synapses between the existing neurons in order to represent new information.
  14. The capacity of computer memory (mainly the hard disks) has been increased dramatically in the last few decades from a few kB to several GB ($10^{9}$ byte), even TB ($10^{12}$ byte).
  15. The memory capacity of a human brain is equivalent to at least $10^{8419}$ modern computers. In other words, the total memory capacity of computers all over the world is far more less than that of a single human brain. (!!!!!surperised at the capacity of our human brain)

Highlights in the paper:

  1. RTPA is based on logic, set, type (the first three for static behaviors descripiton), process aglebra (for dynamic behaviors description)theories.
  2. Algebra is a form of mathematics that simplifies difficult problems by using symbols to represent variables, calculus, and their relations. The process algebra is a set of formal notations and rules for describing algebraic relations of software processes.
  3. RTPA includes foundamental data type, system architectural notations, meta-processes, algebrais relations.
  4. In RTPA a software system is perceived and described mathematically as a set of coherent processes. A process in RTPA is a computational operation that transforms a system from a state to another by changing its inputs, outputs, and/or internal variables.
  5. RTPA predefined 14 meta data types are shown in table 2.
  6. A set of 10 typical and frequently used combinational data objects in system architectural modeling, the abstract data types (ADTs) and their allowable operations, are selected and predefined in RTPA as shown in table 3.
  7. A meta-process is an elementary process that serves as a basic building block in a software system. There are 16 fundamental meta-processes iden- tified in RTPA including (system control, event/time manipulation, memory manipulation, and I/O manipulation processes) as shown in table 1.
  8. Despite of only 16 meta-processes, however,(!!!!)via the combina- tion of a number of meta-processes, any architecture and behavior of software systems can be sufficiently described.
  9. The combination rules of meta-processes in RTPA are governed by a set of algebraic process relations as described in table 4. Any complex process can be combinatory implemented by the basic process relations as shown in table 4.
  10. In the RTPA system specification and refinement scheme as shown in Fig. 1, a new concept, the component logical model (CLM), is introduced, the component logical model (CLM), is introduced, describing the abstract logical models of system hardware and system control mechanisms.
  11. There are four types of system meta-architectures known as: parallel, serial, pipeline, and nested. Any complicated system architecture can be represented by a combination of these four meta-architectures between components. Each of the meta-architectures corresponding to a key RTPA process relation as defined in fig. 2.
